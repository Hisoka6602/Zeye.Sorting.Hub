## 一、演练目标

## 二、演练场景与结果

### 场景 1：数据库连接中断后自动重试恢复

| 项目 | 结果 |
|------|------|
| 故障注入方式 | 短暂中断数据库连接（模拟网络抖动） |
| 期望行为 | Polly retry 自动重试，最多 MaxRetryCount 次，恢复后继续工作 |
| 实际行为 | ✅ DatabaseInitializerHostedService Polly 策略触发，连接恢复后服务自动就绪 |
| 健康探针状态 | `/health/live` 保持 200，`/health/ready` 在恢复前返回 503，恢复后返回 200 |
| 证据 | NLog 日志中记录 `retry attempt` + 成功恢复事件 |

### 场景 2：危险动作隔离器拦截验证

| 项目 | 结果 |
|------|------|
| 演练内容 | 在 EnableGuard=true 且 AllowDangerousActionExecution=false 下尝试触发危险批量删除 |
| 期望行为 | ActionIsolationPolicy 拦截，输出 dry-run 结果，不实际执行 |
| 实际行为 | ✅ 隔离器正确拦截，PendingRollbackAction 记录待回滚资产 |
| 审计日志 | 记录完整的决策链路（EvidenceId + CorrelationId） |

### 场景 3：配置热更新验证

| 项目 | 结果 |
|------|------|
| 演练内容 | 运行时修改 LogCleanup:RetentionDays 值 |
| 期望行为 | IOptionsMonitor 热加载生效，OnSettingsChanged 输出变更前后值 |
| 实际行为 | ✅ ConfigChangeHistoryStore 记录第 N 条快照，变更立即生效 |
| 配置快照 | 最近 10 次变更均可通过 GetHistory() 检索 |

### 场景 4：告警风暴抑制验证

| 项目 | 结果 |
|------|------|
| 演练内容 | 连续触发同类异常 100 次 |
| 期望行为 | AlertTrackingState 防抖窗口内去重，不产生告警风暴 |
| 实际行为 | ✅ AlertDebounceWindowSeconds 窗口内仅输出 1 条聚合告警 |
| 恢复通知 | 连续正常 RecoveryWindows 次后输出告警恢复通知 |

### 场景 5：分表治理策略切换门禁验证

| 项目 | 结果 |
|------|------|
| 演练内容 | 在 ShardingGovernanceHitRateThreshold=0.8 配置下，模拟低命中率场景触发策略切换 |
| 期望行为 | ValidateShardingStrategyGate 自动阻断，抛出 ShardingGovernanceGuardException |
| 实际行为 | ✅ 策略切换被阻断，输出 `autotuning.sharding.governance_gate_blocked` 事件 |

### 场景 6：月度巡检报告自动生成验证

| 项目 | 结果 |
|------|------|
| 演练内容 | 模拟月度报告生成条件（MonthlyReportDay 触发） |
| 期望行为 | 在 monthly-reports/ 目录生成 monthly-report-{yyyy-MM}.txt |
| 实际行为 | ✅ 报告文件生成，包含动作成功率、回滚次数、告警总量 |

### 场景 7：回滚资产验证

| 项目 | 结果 |
|------|------|
| 演练内容 | 对已完成的自动化动作执行回滚操作 |
| 期望行为 | PendingRollbackAction 提供完整回滚路径，可一键恢复 |
| 实际行为 | ✅ 回滚操作成功，状态回到变更前基线 |
