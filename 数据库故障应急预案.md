# 数据库故障应急预案

> 适用范围：数据库连接失败、连接池耗尽、慢查询暴增、迁移失败、归档失败、备份失败、磁盘空间不足  
> 关联实现：`DatabaseConnectionDetailedHealthCheck`、`DatabaseInitializerHostedService`、`MigrationGovernanceHostedService`、`DataArchiveHostedService`、`BackupHostedService`

---

## 一、应急分级

### P1：主链路不可用

- `/health/ready` 持续失败
- 启动失败
- 数据库连接完全不可用
- 迁移阻断导致服务无法提供最小能力

### P2：主链路可用但明显退化

- 数据库连接池耗尽
- 查询 P99 升高
- 慢查询暴增
- 归档任务失败
- 备份失败
- 审计日志过大或磁盘空间不足

---

## 二、统一处置原则

1. 先保只读、后恢复写入、最后恢复后台治理任务。
2. 先确认健康检查和日志证据，再执行重试、切流、清理、迁移、补偿。
3. 涉及危险动作时必须保留 dry-run 结果、审计记录、回滚资产路径。
4. 所有故障处置结束后，必须回填到 `drill-records/` 或对应更新记录。

---

## 三、故障处理剧本

### 1. 数据库连接失败

1. 检查 `/health/ready` 与 `DatabaseConnectionDetailedHealthCheck` 输出。
2. 核对 `Persistence:Provider` 与对应连接字符串配置键是否存在。
3. 检查数据库服务状态、网络连通性、DNS、端口、凭据占位符是否被错误替换。
4. 若为短暂抖动，允许依赖 `DatabaseInitializerHostedService` 内重试链路自动恢复。
5. 若恢复后健康检查仍异常，立即切到人工值守模式，暂停变更。

### 2. 数据库连接池耗尽

1. 检查当前慢查询画像、长事务、只读副本路由结果。
2. 观察 `ResourceThresholds` 配置是否已触发连接池阈值告警。
3. 先抑制报表型读请求，再检查批量缓冲写入和归档任务是否放大压力。
4. 若只读副本健康，优先将报表查询切到只读路径。

### 3. 慢查询暴增 / 查询 P99 升高

1. 使用 `/api/diagnostics/slow-queries` 导出当前画像。
2. 检查 `QueryGovernanceReportHostedService` 输出与索引建议闭环。
3. 核对 `性能基线报告.md` 的最近基线，确认异常偏离程度。
4. 对未登记模板、缺索引、跨表查询占比过高的链路优先处理。

### 4. 迁移失败

1. 检查 `MigrationGovernanceHostedService` 与 `MigrationGovernanceHealthCheck`。
2. 核对归档脚本、待执行迁移、危险 SQL 判定结果。
3. 若存在危险 SQL，禁止跳过治理直接上线。
4. 需要回退时，优先使用迁移归档资产与既有回滚脚本。

### 5. 归档任务失败

1. 查询 `/api/data-governance/archive-tasks` 的失败任务与 dry-run 结果。
2. 检查 `DataArchiveHostedService` 后台执行日志。
3. 先重试 dry-run，确认失败原因是窗口、表状态、容量还是 SQL 执行异常。
4. 故障未排清前，不允许直接删除失败任务记录。

### 6. 备份失败

1. 检查 `BackupHealthCheck` 的最新备份文件路径、Runbook 路径、演练记录路径。
2. 检查 `BackupHostedService` 与 `BackupVerificationService` 日志。
3. 确认磁盘容量、备份目录权限、Provider 命令生成是否正常。
4. 恢复后按 `备份恢复演练Runbook.md` 完成一次恢复验证。

### 7. 磁盘空间不足 / 审计日志过大

1. 先确认 NLog 日志目录、备份目录、归档目录的占用情况。
2. 检查 `DataRetentionHostedService` 是否按周期执行。
3. 检查审计日志 API 使用量、保留策略与归档结果。
4. 仅允许按既定保留策略清理，禁止直接删除最近窗口内的关键证据文件。

---

## 四、恢复确认

1. `/health/live`、`/health/ready` 恢复正常。
2. 核心只读接口、写入接口、归档任务、备份健康检查恢复正常。
3. 慢查询画像和 P99 指标回落到基线附近。
4. 更新 `业务接入前底座验收清单.md` 的相关证据，必要时补充演练记录。
